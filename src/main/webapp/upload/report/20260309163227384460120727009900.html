예외 처리
1
CLIENT POST /auth/login 요청 login_id + password
클라이언트가 아이디와 비밀번호를 담아 로그인 요청을 전송합니다.
// 요청 Body 예시 { "login_id": "user123", "password": "mypassword" }
2
SERVICE CustomUserDetailsService.loadUserByUsername() CustomUserDetailsService.java
MemberService를 통해 DB에서 사용자를 조회하고 권한 정보를 가져옵니다. 결과를 CustomUserDetails로 래핑하여 반환합니다.
public UserDetails loadUserByUsername(String username) { MemberDto member = memberService.findById(username); if (member == null) throw new UsernameNotFoundException(...); String authority = memberService.findByAuthority(username); // authorities → CustomUserDetails 빌드 return CustomUserDetails.builder() .loginUser(loginUser) .roles(authorities) .build(); }
3
DOMAIN NumericRoleGranted.getUserLevel() 권한 매핑 NumericRoleGranted.java
문자열 권한(USER, EMP, ADMIN 등)을 숫자 레벨로 변환하여 levelCode에 저장. 이 값을 기반으로 주간보고서 등 role-based 접근 제어가 동작합니다.
INACTIVE = 0 // 비회원 USER = 1 // 일반회원 INSTRUCTOR= 31 // 강사 EX_EMP = 50 // 퇴사사원 EMP = 51 // 사원 → levelCode >= 51: 팀 보고서 조회 가능 ADMIN = 99 // 관리자 → 전체 접근
4
JWT JwtTokenProvider.generateToken() JwtTokenProvider.java
인증된 Authentication 객체로 AccessToken(24시간)과 RefreshToken(7일)을 생성합니다. HMAC SHA 알고리즘으로 서명합니다.
// Access Token: subject(login_id) + role 클레임 포함 String accessToken = Jwts.builder() .subject(authentication.getName()) .claim("role", authorities) .expiration(new Date(now + 86400000L)) // 24h .signWith(secretKey) .compact(); // Refresh Token: subject 없이 만료 기간만 String refreshToken = Jwts.builder() .expiration(new Date(now + 604800000L)) // 7d .signWith(secretKey) .compact();
5
RESPONSE JwtToken DTO 반환 JwtToken.java
클라이언트에 grantType, accessToken, refreshToken, 만료시간을 JSON으로 전달합니다.
{ "grantType": "Bearer", "accessToken": "eyJhbGciOiJIUzI1...", "accessTokenExpiresIn": 1741234567890, "refreshToken": "eyJhbGciOiJIUzI1..." }

각 단계를 클릭하면 코드 상세를 확인할 수 있습니다

1
CLIENT API 요청 (Authorization 헤더 포함) HTTP Header
클라이언트는 모든 API 요청 헤더에 Bearer {accessToken}을 담아 전송합니다.
GET /api/weekly-report HTTP/1.1 Authorization: Bearer eyJhbGciOiJIUzI1NiJ9... Content-Type: application/json
2
FILTER JwtAuthenticationFilter.doFilterInternal() JwtAuthenticationFilter.java
OncePerRequestFilter 상속. 요청마다 딱 한 번 실행됩니다.
resolveToken()으로 헤더에서 Bearer 뒤의 토큰 문자열을 추출합니다.
// "Bearer eyJhbGc..." → "eyJhbGc..." 추출 private String resolveToken(HttpServletRequest request) { String bearerToken = request.getHeader("Authorization"); if (StringUtils.hasText(bearerToken) && bearerToken.startsWith("Bearer ")) { return bearerToken.split(" ")[1].trim(); } return null; }
3
JWT JwtTokenProvider.validateToken() JwtTokenProvider.java
secretKey로 서명을 검증합니다. 만료·위변조·형식오류 발생 시 각각 다른 예외를 던집니다.
Jwts.parser() .verifyWith(secretKey) .build() .parseSignedClaims(token); // 서명 검증 // 예외 분기 catch (ExpiredJwtException e) → throw e // 만료 catch (MalformedJwtException e) → throw JwtException // 위변조 catch (UnsupportedJwtException) → throw JwtException // 미지원 catch (IllegalArgumentException) → throw JwtException // 빈토큰
4
JWT JwtTokenProvider.getAuthentication() JwtTokenProvider.java
토큰 Claims에서 subject(login_id)를 꺼내고, 다시 DB에서 사용자 정보를 로드하여 UsernamePasswordAuthenticationToken을 생성합니다.
Claims claims = parseClaims(accessToken); String username = claims.getSubject(); // login_id // DB 재조회 → 최신 권한 반영 UserDetails userDetails = customUserDetailsService.loadUserByUsername(username); return new UsernamePasswordAuthenticationToken( userDetails, null, userDetails.getAuthorities() );
5
CONTEXT SecurityContextHolder에 Authentication 저장 JwtAuthenticationFilter.java
인증 객체를 SecurityContext에 저장합니다. 이후 컨트롤러에서 @AuthenticationPrincipal이나 @SessionAttribute로 사용자 정보를 꺼낼 수 있습니다.
SecurityContextHolder.getContext() .setAuthentication(authentication); // 이후 filterChain.doFilter() → 컨트롤러 진입 filterChain.doFilter(request, response);
6
SUCCESS Controller 진입 및 비즈니스 로직 실행 200 OK
모든 검증 통과. 컨트롤러에서 levelCode 기반의 분기 처리가 가능합니다.
// 예: 주간보고서 컨트롤러 SessionInfo member = (SessionInfo) session.getAttribute("member"); if (member.getLevelCode() < 51) { // 본인 보고서만 조회 } else if (member.getLevelCode() == 99) { // 전체 조회 }
!
ERROR 예외 처리 핸들러 EntryPoint / AccessDenied
401 Unauthorized → JwtAuthenticationEntryPoint (미인증 접근)
403 Forbidden → JwtAccessDeniedHandler (권한 부족)
토큰 만료/위변조 → Filter에서 직접 JSON 에러 응답
// 만료 → 401 JSON 응답 catch (ExpiredJwtException e) { setErrorResponse(response, "토큰이 만료되었습니다."); } // 권한 없음 → 403 response.sendError(HttpServletResponse.SC_FORBIDDEN);
클래스 구성 및 역할
🔍
JwtAuthenticationFilter
extends OncePerRequestFilter
모든 HTTP 요청에서 JWT를 추출·검증하고 SecurityContext에 인증 정보를 세팅하는 핵심 필터. 요청당 정확히 1회 실행 보장.
doFilterInternal() resolveToken() setErrorResponse()
⚙️
JwtTokenProvider
@Component
JWT의 생성·복호화·검증을 담당. HMAC-SHA 알고리즘 서명. AccessToken(24h) + RefreshToken(7d) 이중 구조.
generateToken() getAuthentication() validateToken() parseClaims()
👤
CustomUserDetailsService
implements UserDetailsService
DB에서 사용자 정보를 조회하여 CustomUserDetails로 래핑. Spring Security 인증 흐름의 진입점.
loadUserByUsername() toUserDetails()
🪪
CustomUserDetails
implements UserDetails · Builder 패턴
Spring Security의 UserDetails 구현체. Builder 패턴으로 생성. LoginUser DTO + 권한 목록 + 활성화 여부 포함.
getAuthorities() getUsername() isEnabled() getLoginUser()
🔢
NumericRoleGranted
유틸리티 클래스
문자열 권한(USER, EMP, ADMIN)을 숫자 levelCode로 매핑. 주간보고서 등 분기 처리에 사용 (0/1/31/50/51/99).
getUserLevel(String)

JwtToken
DTO · @Builder @Data
로그인 성공 시 클라이언트에 반환하는 응답 DTO. grantType, accessToken, refreshToken, 만료시간(epoch ms) 포함.
grantType accessToken refreshToken expiresIn
